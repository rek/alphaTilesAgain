## ADDED Requirements

### Requirement: `yue` pack ships a smoke-test batch of composite numerals as wordlist entries

The `yue` language pack SHALL include the 5 composite numerals `二十`, `三十`, `五十`, `一百萬`, `二百零三` as wordlist rows with matching Arabic-digit reference images, but SHALL NOT add corresponding tile rows (composites decompose into existing single-character tiles).

#### Scenario: Composite wordlist rows are appended

- **GIVEN** the 5 composite numerals
- **WHEN** `languages/yue/aa_wordlist.txt` is parsed by `parseWordlist`
- **THEN** the row count increases by 5 over the pre-change baseline
- **AND** each composite character sequence appears exactly once as a `wordInLOP` value
- **AND** the matching `wordInLWC` values are `zz_20`, `zz_30`, `zz_50`, `zz_1000000`, `zz_203` respectively

#### Scenario: No new tile rows are added

- **WHEN** the change's diff against `languages/yue/aa_gametiles.txt` is inspected
- **THEN** the file is byte-identical to its pre-change state
- **AND** each composite's constituent characters (`二`, `三`, `五`, `十`, `百`, `萬`, `零`) are already present as single-character tile rows from `yue-numerals-game`

#### Scenario: Composite LOPs fully decompose into known syllables

- **GIVEN** the 5 composite wordlist rows
- **WHEN** `parseWordIntoSyllables(wordInLOP, assets.syllables.rows)` is called for each
- **THEN** every call returns a non-empty array of syllable rows whose concatenation equals `wordInLOP`
- **AND** no composite triggers the empty-array (unparseable) result

### Requirement: Arabic-digit reference images exist for every composite

Each composite wordlist row SHALL have an `images/words/<lwc>.png` rendering its Arabic-digit equivalent, generated by extending `tools/build-numeral-images.ts`.

#### Scenario: Images are present after running the build tool

- **WHEN** `npx tsx tools/build-numeral-images.ts` is run
- **THEN** `languages/yue/images/words/zz_20.png`, `zz_30.png`, `zz_50.png`, `zz_1000000.png`, and `zz_203.png` exist and are non-empty
- **AND** each PNG renders the corresponding Arabic-digit label (`20`, `30`, `50`, `1000000`, `203`)
- **AND** the tool's `NUMERALS` constant contains entries for each composite LWC

#### Scenario: Validator does not report missing word images

- **WHEN** the lang-pack validator runs `checkImageReferences` against the `yue` pack
- **THEN** no `MISSING_WORD_IMAGE` issue is raised for any composite LWC
- **AND** no `ORPHAN_IMAGE` issue is raised for the new composite PNGs

### Requirement: Composites ship without per-word audio recordings

The composite wordlist rows SHALL NOT have corresponding `audio/words/<lwc>.mp3` files; runtime audio for these words is the shell's syllable-chain fallback (see `game-engine` capability delta).

#### Scenario: Validator surfaces missing word audio as warning only

- **GIVEN** `aa_settings.txt` has `Has word audio: TRUE` and 5 composite rows have no `audio/words/<lwc>.mp3`
- **WHEN** the lang-pack validator runs `checkAudioReferences` against the `yue` pack
- **THEN** 5 `MISSING_WORD_AUDIO` issues are emitted as warnings (severity ≤ `WARNING`)
- **AND** the validator's exit summary reports `0 error(s)`

#### Scenario: Future single recordings drop in transparently

- **GIVEN** a composite row `zz_20` with no `audio/words/zz_20.mp3`
- **WHEN** Curtis ships a single recording and the file is copied to `audio/words/zz_20.mp3`
- **AND** the manifest is regenerated and the app boots
- **THEN** `audio.handles.words.get('zz_20')` resolves to a non-null sound handle
- **AND** the shell's `replayWord` plays the single recording, NOT the syllable chain

### Requirement: No engine code or game-class is added

This change SHALL modify only pack data, the `tools/build-numeral-images.ts` script, the `feature-game-shell` audio fallback, and the location of `parseWordIntoSyllables` (move from `feature-game-georgia` to `util-phoneme`). No new `feature-game-*` lib, no new doors in `aa_games.txt`, no new `data-*` lib.

#### Scenario: No new game-feature libraries

- **WHEN** the change's diff is inspected
- **THEN** no new directory is created under `libs/alphaTiles/feature-game-*`
- **AND** `aa_games.txt` is byte-identical to its pre-change state

#### Scenario: `data-audio` is unchanged

- **WHEN** the change's diff is inspected
- **THEN** no file under `libs/alphaTiles/data-audio/src/` is modified
