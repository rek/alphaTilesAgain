## Why

Issue #28 asked for composite numeral games (二十, 一百萬, 二百零三, …) on top of the 13 base numerals shipped in `yue-numerals-game` + `fe1bcdc`. The customer (Curtis-LeadM) asked whether composites need per-word recordings or can be assembled at runtime from existing per-character audio. We settled on a middle path on the issue thread: ship composites now using runtime syllable-by-syllable playback as the v1 audio (pedagogically clearer than smoothed concat; silent gap = teaching cue), and let single recordings transparently swap in later when they arrive. This change proves that path end-to-end with a small smoke-test batch.

## What Changes

- Add 5 composite numeral words to the `yue` pack: `二十` (zz_20), `三十` (zz_30), `五十` (zz_50), `一百萬` (zz_1000000), `二百零三` (zz_203). No new tiles — composites decompose into existing single-character tiles.
- Generate Arabic-digit reference images for the 5 composites via `tools/build-numeral-images.ts`.
- Add a **syllable-chain audio fallback** in `data-audio`: when `playWord(lwc)` is called for a word without per-word audio, decompose its LOP into syllables, play each syllable's existing clip in sequence with a short natural pause between (timer-ref pattern, not chained promises — per `docs/GAME_PATTERNS.md`).
- The fallback hooks at the audio layer, so every game that calls `playWord` (Thailand doors 1–5, Georgia T-mode doors 10–11) picks up composite playback automatically without container changes.
- No engine refactor, no new doors, no new tiles, no new game classes.

## Capabilities

### New Capabilities
- `yue-composite-numerals`: pack-data additions for 5 composite numerals, the Arabic-digit image generation, and the validator-clean state when per-word audio is absent for composites.

### Modified Capabilities
- `game-engine`: the shell's audio-replay path SHALL fall back to a sequential syllable chain when the active `refWord` has no per-word audio handle but its LOP decomposes into known syllables. Lock/unlock semantics around the chain are unchanged from the single-clip path.
