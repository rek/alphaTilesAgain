## Context

`Peru.java` (~250 LOC) is a 4-choice image-to-word recognition game. The `challengeLevel` controls how the 3 wrong answers are generated by mutating tiles in the correct word. There is no syllable variant.

## Goals / Non-Goals

**Goals:**
- Match Java behaviour exactly across CL1/CL2/CL3, **including the `nextInt(tileLength - 1)` index quirk** that excludes the last tile from random replacement (CL2/CL3).
- Match Java's CL1 behaviour: 3 wrong answers come from **the 3 distinct distractors of the first tile** (no regen loop required); only the ўДўДџБ filter applies.
- Container/Presenter split; i18n-blind presenter; no `useEffect` outside the mount kickoff pattern.
- Unit-test the pure logic functions against Java behaviour with a seeded RNG.

**Non-Goals:**
- Syllable variant.
- Arabic-ligature filter beyond `ўДўДџБ`.
- Distractor types beyond same-type and distractor-trio.
- Animations.

### D2. Challenge-level decoding

### D3. Wrong-Answer Generation per CL (matches Java)

```ts
// CL1 вАФ match Peru.java:127вАУ143
//   THE WRONG ANSWERS ARE LIKE THE RIGHT ANSWER EXCEPT HAVE ONLY ONE TILE (THE FIRST TILE) REPLACED
//   REPLACEMENT IS FROM DISTRACTOR TRIO (one trio entry per wrong slot)
function buildWrongCL1(parsed: ParsedTile[], trioShuffled: TileEntry[], slotIndex: 0|1|2): {
  text: string;
} {
  const replacement = trioShuffled[slotIndex];
  const tiles = [...parsed];
  tiles[0] = toParsedTile(replacement, parsed[0].typeOfThisTileInstance);
  const text = combineTilesToMakeWord(tiles, refWord.wordInLOP, 0, scriptType);
  return { text };
  // NOTE: ўДўДџБ rejection regenerates by retrying with the SAME slot via fallback; in
  // practice the trio is finite (3 entries). We log + retry by drawing one of the
  // remaining trio entries; if all 3 contain ўДўДџБ we fall back to a CL3-style draw.
}
```

```ts
// CL2 вАФ match Peru.java:146вАУ175
//   ANY TILE OF THE SAME TYPE (C OR V OR T OR AD) FROM THE WHOLE ARRAY
//   randIdx = rand.nextInt(tileLength - 1)  // вЖР Java excludes last tile (kept for parity)
function buildWrongCL2(parsed, prior, correct, pools, rng): { text: string } | null {
  for (let guard = 0; guard < 200; guard++) {
    const idx = Math.floor(rng() * (parsed.length - 1));
    const sameType = pools.byType(parsed[idx].typeOfThisTileInstance); // V/C/T/AD
    if (!sameType?.length) continue;
    const replacement = sameType[Math.floor(rng() * sameType.length)];
    const tiles = [...parsed];
    tiles[idx] = toParsedTile(replacement, parsed[idx].typeOfThisTileInstance);
    const text = combineTilesToMakeWord(tiles, refWord.wordInLOP, idx, scriptType);
    if (containsForbidden(text)) continue;
    if (text === correct) continue;
    if (prior.includes(text)) continue;
    return { text };
  }
  return null;
}
```

```ts
// CL3 вАФ match Peru.java:177вАУ204
//   ONE RANDOM TILE REPLACED BY ANY DISTRACTOR OF THAT TILE
//   randIdx = rand.nextInt(tileLength - 1)  // вЖР same off-by-one as CL2 (kept)
function buildWrongCL3(parsed, prior, correct, tileMap, rng): { text: string } | null {
  for (let guard = 0; guard < 200; guard++) {
    const idx = Math.floor(rng() * (parsed.length - 1));
    const trio = tileMap.get(parsed[idx].base);
    if (!trio) continue;
    const trioArr = [trio.alt1, trio.alt2, trio.alt3].filter((b) => !!b && tileMap.has(b));
    if (!trioArr.length) continue;
    const replacement = tileMap.get(trioArr[Math.floor(rng() * trioArr.length)])!;
    const tiles = [...parsed];
    tiles[idx] = toParsedTile(replacement, parsed[idx].typeOfThisTileInstance);
    const text = combineTilesToMakeWord(tiles, refWord.wordInLOP, idx, scriptType);
    if (containsForbidden(text)) continue;
    if (text === correct) continue;
    if (prior.includes(text)) continue;
    return { text };
  }
  return null;
}
```

### D4. Top-level choice assembly

```ts
const FORBIDDEN = 'ўДўДџБ';
function containsForbidden(s: string): boolean { return s.includes(FORBIDDEN); }

function buildAllChoices({ level, parsed, refWord, tileMap, pools, scriptType, rng }): {
  correct: string;
  choices: string[]; // length 4, in random slot order, contains correct
} | { error: 'degenerate' } {
  const correct = standardizeWordSequence({ wordInLOP: refWord.wordInLOP, ... });
  const slot = Math.floor(rng() * 4);
  const wrongs: string[] = [];

  if (level === 1) {
    // Java: shuffle the trio once, then take entries 0,1,2 in order
    const trioShuffled = shuffle([alt1, alt2, alt3].map(b => tileMap.get(b)!).filter(Boolean), rng);
    for (let i = 0; i < 3; i++) {
      const replacement = trioShuffled[i] ?? trioShuffled[i % trioShuffled.length];
      const text = combineTilesToMakeWord(...);
      if (containsForbidden(text)) {
        // Per Java: while-loop just retries SAME slot вАФ try other trio entries; if exhausted, mark degenerate.
        ...
      }
      wrongs.push(text);
    }
  } else {
    for (let i = 0; i < 3; i++) {
      const w = (level === 2 ? buildWrongCL2 : buildWrongCL3)(parsed, [correct, ...wrongs], correct, ...);
      if (!w) return { error: 'degenerate' };
      wrongs.push(w.text);
    }
  }

  const choices = [...wrongs];
  choices.splice(slot, 0, correct);
  return { correct, choices };
}
```

If `degenerate`, container picks another `refWord` (max 5 retries вЖТ insufficient-content error screen).

### D5. Image Tap = Replay Word Audio

Tapping the word image MUST replay the active word clip via `shell.replayWord()` (which calls `audio.playWord(refWord.wordInLWC)`). Java: `clickPicHearAudio` вЖТ super's `playActiveWordClip`.

### D6. Container / Presenter Split

**`<PeruContainer>`** (wraps `<GameShellContainer>`) вЖТ inner `<PeruGame>` owns state via `useGameShell()`:
- Hooks: `useGameShell`, `useLangAssets`, `useAudio`.
- State: `correctText`, `choices: { text; grayed; bgColor }[]`, `error: 'insufficient-content' | null`.
- Refs: `isMountedRef` (cleanup guard).
- Handlers: `onChoicePress(index)`, `onImagePress()` (= `shell.replayWord`).
- `useMountEffect`: kickoff `playAgain()`; register `shell.setOnAdvance(playAgain)`.

**`<PeruScreen>`** вАФ pure props вЖТ JSX:
- Props: `wordImage: ImageSourcePropType | undefined`, `wordLabel: string`, `choices: { text; grayed; bgColor }[]`, `interactionLocked`, `onChoicePress(i)`, `onImagePress()`.
- 2√Ч2 grid of choice cells; each cell is full button (Pressable) coloured from `colors.hexByIndex[i]`.
- No hooks beyond `useWindowDimensions` for sizing; no i18n.

### D7. Insufficient-content fallback

If a chosen word can't be parsed, has < 2 tiles (CL2/CL3 require `tileLength - 1 >= 1`), or generation degenerates after 5 attempts, render an empty/locked screen. Match china's pattern: `setError('insufficient-content')`, render disabled `<PeruScreen>`.

### D9. RTL handling

Container/screen do not flip layout manually. Logical props (`marginStart`, `gap`) are RTL-aware. Java's `fixConstraintsRTL` is N/A.

## Unresolved Questions

1. **None blocking.** Java's `nextInt(tileLength - 1)` off-by-one IS preserved (per user direction "match java"). If the team later wants to fix the bug, change `len - 1` вЖТ `len` in CL2/CL3 and update tests.
2. ўДўДџБ filter on CL1 is rare in practice (only fires when a word's first-tile distractor produces the substring). Java just keeps looping with the *same* trio entry вАФ we extend by trying remaining trio entries before falling back to insufficient-content.
